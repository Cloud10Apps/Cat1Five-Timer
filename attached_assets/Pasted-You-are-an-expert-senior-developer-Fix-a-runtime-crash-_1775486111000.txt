You are an expert senior developer. Fix a runtime crash in the inspection 
edit form that occurs when the user changes the year in the completion 
date picker dropdown.

THE CAUSE (confirmed by analysis):
The DatePickerField fires onChange with intermediate invalid date strings 
during year dropdown changes (e.g. "2025-04-0", "202", etc.). These 
partial strings pass a length check but fail dayjs parsing, causing 
.year() to return NaN which crashes the React render.

The fix must use dayjs().isValid() as the guard — NOT a string length 
check — because intermediate YYYY-MM-DD strings can be 10 characters 
long but still be invalid dates.

Replace it with this more robust version that strictly validates 
the YYYY-MM-DD format before parsing:
  function isValidDateStr(value: string | undefined): boolean {
    if (!value) return false;
    // Must match exactly YYYY-MM-DD with valid ranges
    const match = value.match(/^(\d{4})-(\d{2})-(\d{2})$/);
    if (!match) return false;
    const year  = parseInt(match[1], 10);
    const month = parseInt(match[2], 10);
    const day   = parseInt(match[3], 10);
    if (year < 1900 || year > 2200) return false;
    if (month < 1   || month > 12)  return false;
    if (day < 1     || day > 31)    return false;
    // Final check — let dayjs confirm it's a real calendar date
    return dayjs(value, "YYYY-MM-DD", true).isValid();
  }
