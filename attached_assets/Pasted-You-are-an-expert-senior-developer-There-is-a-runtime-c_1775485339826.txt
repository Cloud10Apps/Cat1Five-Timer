You are an expert senior developer. There is a runtime crash in the 
Current Inspections by Unit page (elevators.tsx) that occurs when a 
user types a completion date character by character into the date field.

FILE: artifacts/elevator-tracker/src/pages/elevators.tsx

THE CAUSE:
When the user types into the completion date field, the intermediate 
partial string (e.g. "04", "04/0", "04/06") is passed to dayjs() 
before the date is complete. dayjs() returns an invalid date for these 
partial strings, and calling .year() on an invalid dayjs object returns 
NaN, which causes a runtime error when rendered in the JSX.

THE FIX — Two changes needed:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CHANGE 1: Add an isValidDate helper function
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Near the top of the file, after the imports, add this helper function:

  function isValidDateStr(value: string | undefined): boolean {
    if (!value || value.length < 10) return false;
    const d = dayjs(value);
    return d.isValid();
  }
